## 시작하기

### 1. 설정 페이지에서 API 키 설정

Chat Bot을 사용하려면 먼저 설정이 필요합니다:

1. 메인 페이지에서 **설정** 클릭
2. **AI API** 탭에서 Claude 또는 Gemini API 키 입력
3. **Confluence** 탭에서 Atlassian 인증 정보 입력 (선택)
4. **Jira** 탭에서 Jira 인증 정보 입력 (선택)

### 2. Chat Bot 페이지 접속

메인 페이지에서 **Chat Bot** 카드를 클릭하거나 사이드바에서 **Chat Bot**을 선택합니다.

## 기본 사용법

### 질문하기

입력창에 질문을 입력하고 **전송** 버튼을 클릭하거나 `Enter` 키를 누릅니다.

```
예시: "AEGIS 프로젝트의 아키텍처에 대해 알려줘"
```

### AI 모델 선택

입력창 옆의 드롭다운에서 사용할 AI 모델을 선택할 수 있습니다:
- **Claude** (Anthropic) - 복잡한 분석과 긴 문서 처리에 강함
- **Gemini** (Google) - 빠른 응답과 다국어 지원

### 대화 기록

- 왼쪽 사이드바에서 이전 대화 기록을 확인할 수 있습니다
- 대화를 클릭하면 해당 대화로 이동합니다
- **새 대화** 버튼으로 새로운 대화를 시작합니다

## Confluence 문서 검색

Chat Bot은 Confluence 문서를 자동으로 검색하여 답변에 활용합니다.

### 문서 검색 예시

```
"봇 AI 설계 문서 찾아줘"
"사격 판단 로직에 대한 문서가 있어?"
"AEGIS 아키텍처 관련 문서 보여줘"
```

### 문서 목록 보기

```
"컨플루언스 문서 목록 보여줘"
"컨플 문서 뭐 있어?"
"최근 문서 알려줘"
```

### 특정 주제 검색

```
"벤치마크 관련 문서 찾아줘"
"AI 행동 트리 문서 있어?"
"네트워크 동기화 관련 내용 알려줘"
```

## Jira 이슈 검색

Jira 이슈를 다양한 조건으로 검색할 수 있습니다.

### 기본 검색

```
"지라 이슈 보여줘"
"최근 일감 목록 알려줘"
"AEGIS 프로젝트 티켓 찾아줘"
```

### 담당자별 검색

```
"이래관의 일감 보여줘"
"신동효님 담당 이슈 알려줘"
"담당자 홍길동 일감 찾아줘"
```

### 상태별 검색

```
"진행중인 이슈 보여줘"
"완료된 일감 목록"
"대기 상태 티켓 알려줘"
```

### 유형별 검색

```
"버그 이슈 찾아줘"
"태스크 목록 보여줘"
"스토리 일감 알려줘"
"에픽 목록"
```

### 우선순위별 검색

```
"긴급 이슈 보여줘"
"높은 우선순위 일감"
"낮은 우선순위 티켓"
```

### 스프린트 검색

```
"현재 스프린트 이슈 보여줘"
"스프린트 5 일감 알려줘"
"이번 스프린트 진행 상황"
```

### 기한별 검색

```
"오늘 마감 이슈 보여줘"
"이번 주 마감 일감"
"기한 지난 티켓 찾아줘"
```

### 특정 이슈 조회

```
"AEGIS-514 이슈 정보 알려줘"
"AEGIS-123 상세 내용 보여줘"
```

## 고급 검색 기능

### 복합 조건 검색

여러 조건을 조합하여 검색할 수 있습니다:

```
"이래관의 진행중인 버그 이슈 보여줘"
"높은 우선순위 태스크 중 이번 주 마감인 것"
"스프린트 5의 완료된 스토리 목록"
```

### 분석 요청

AI에게 분석을 요청할 수 있습니다:

```
"이 문서들의 연관성을 분석해줘"
"버그 발생 원인을 추론해줘"
"두 설계 방식의 차이점을 비교해줘"
"이 문제의 해결책을 제안해줘"
```

### 스프린트/마일스톤 정보

```
"현재 스프린트 정보 알려줘"
"스프린트 목표가 뭐야?"
"마일스톤 일정 보여줘"
"릴리스 계획 알려줘"
```

> **Sprint 5** (2024.01.15 ~ 2024.01.29)
> - 남은 기간: 7일
> - 목표: AI 행동 트리 개선
> 
> ### 주요 이슈
> | 이슈 | 상태 | 담당자 |
> |------|------|--------|
> | AEGIS-514 | 진행중 | 이래관 |
> | AEGIS-515 | 대기 | 신동효 |
> ...

### 예제 4: 버그 원인 분석

**질문:**
```
"AEGIS-514 버그의 원인을 분석해줘"
```

**AI 응답 예시:**

### 예제 5: 문서 비교 분석

**질문:**
```
"행동 트리와 상태 머신 방식의 차이점을 비교해줘"
```

**AI 응답 예시:**
> ## 행동 트리 vs 상태 머신 비교
> 
> | 항목 | 행동 트리 | 상태 머신 |
> |------|----------|----------|
> | 구조 | 트리 형태 | 그래프 형태 |
> | 확장성 | 높음 | 중간 |
> | 디버깅 | 용이 | 복잡 |
> | 성능 | 중간 | 높음 |
> 
> ### 권장 사용 케이스
> - **행동 트리**: 복잡한 AI 행동, 모듈화 필요 시
> - **상태 머신**: 단순한 상태 전이, 성능 중요 시
> 
> 참고: [AI 설계 문서](link)

## 팁과 트릭

### 1. 구체적으로 질문하기

❌ 나쁜 예: `"문서 찾아줘"`

✅ 좋은 예: `"AI 봇의 사격 판단 로직에 대한 문서 찾아줘"`

### 2. 한국어/영어 혼용 가능

```
"AEGIS 프로젝트의 architecture 문서 찾아줘"
"bug 이슈 중 진행중인 것 보여줘"
```

### 3. 자연스러운 대화체 사용

```
"이래관님이 담당하는 일감이 뭐가 있어?"
"지금 진행중인 스프린트 목표가 뭐야?"
"이 버그 왜 발생했을까?"
```

### 4. 후속 질문하기

이전 대화 맥락을 활용하여 후속 질문을 할 수 있습니다:

```
1번째 질문: "AEGIS-514 이슈 정보 알려줘"
2번째 질문: "이 이슈의 원인을 분석해줘"
3번째 질문: "해결책을 제안해줘"
```

### 5. 참조 문서 활용

AI 응답 아래에 표시되는 **참조 문서**를 클릭하면 원본 문서로 이동할 수 있습니다.

## 문제 해결

### API 키 오류

> "API 키가 설정되지 않았습니다"

→ 설정 페이지에서 Claude 또는 Gemini API 키를 입력하세요.

### Confluence 연동 오류

> "Confluence 문서를 찾을 수 없습니다"

→ 설정 페이지에서 Confluence 인증 정보를 확인하세요.

### Jira 연동 오류

> "Jira 이슈를 검색할 수 없습니다"

→ 설정 페이지에서 Jira 인증 정보를 확인하세요.

### 느린 응답

- AI 모델을 Gemini로 변경해보세요 (더 빠름)
- 질문을 더 구체적으로 작성하세요
